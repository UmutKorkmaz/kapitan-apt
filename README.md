# kapitan-apt

Signed **apt repository** for [KAPiTaN OS](https://github.com/UmutKorkmaz/KAPiTaN-OS),
served via GitHub Pages at:

```
https://umutkorkmaz.github.io/kapitan-apt/
```

## Why a separate repo?

The main KAPiTaN-OS repo already publishes its **website** to GitHub Pages. A
GitHub Pages deploy replaces the *entire* site artifact, so hosting the apt repo
on the same Pages site would atomically wipe the website (and vice versa) on
every deploy — breaking `apt update` for every user. The apt repository
therefore lives here, in its own Pages site. (See decision **A4** in the
KAPiTaN-OS build plan.)

## What's in here

| Path | Purpose |
|------|---------|
| `conf/distributions` | reprepro suite definition (`alpha`); `SignWith` = signing subkey fingerprint. |
| `conf/options` | reprepro global options. |
| `kapitan-archive-keyring.asc` | **Public** signing key, served at the repo root and packaged for users. |
| `incoming/` | Drop zone for `.deb`s before reprepro ingests them (kept by `.gitkeep`). |
| `.github/workflows/publish.yml` | Verifies signed `.deb`s from a KAPiTaN-OS release, ingests them with reprepro, and publishes `pool/` then `dists/` to Pages. |
| `pool/`, `dists/`, `db/` | Generated by reprepro — committed so publishes stay incremental. |

## How publishing works

1. KAPiTaN-OS CI builds and **signs** `kapitan-sh`, `pazar`, and `kapitan-ai`
   `.deb`s, then creates a GitHub Release with the signed assets.
2. That workflow fires a `repository_dispatch` (`kapitan-release-published`) at
   this repo.
3. `publish.yml` downloads the release assets, **verifies** every signature
   against `kapitan-archive-keyring.asc`, ingests them via `reprepro`, and
   publishes — `pool/` before `dists/` to avoid a CDN hash-mismatch window.

## Trust / security

- The repository is **signed** with a CI signing **subkey**. The offline primary
  key never touches CI.
- Users add trust by fetching `kapitan-archive-keyring.asc` over HTTPS (pinned by
  fingerprint) — see
  [`docs/INSTALL-APT.md`](https://github.com/UmutKorkmaz/KAPiTaN-OS/blob/main/docs/INSTALL-APT.md)
  in the main repo.
- Signing-key custody and rotation are documented in
  [`SECURITY.md`](https://github.com/UmutKorkmaz/KAPiTaN-OS/blob/main/SECURITY.md).

## Status

**alpha** — single `alpha` codename, `amd64` + `source`. At beta this moves to
`repo.kapitan-os.org` (Cloudflare, `no-cache` on `dists/`).
